2 ========================================
3 A C++14 reflection markup library, originally developed for use in small-scale custom
4 game engines to help automate tasks like memory management, script binding, serialization,
5 and communication.

9 ----------------------------------------
10 ### Documenting a type
11 
12 The type declaration:
13 
14 ```cpp
15 struct Object {
16  std::string name;
17  Object *children;
18  unsigned child_count;
19 
20  Object& GetChild(int idx);
21 }
22 ```
23 
24 In a .cpp file:
25 
26 ```cpp
27 rkType(Object, rkDefaultFactory
28  rkMember(name)
29  rkMember(children)
30  rkMember(child_count)
31 
32  rkMethod(GetChild)
33 
34  rkInlineGetSet(
35  ObjName,
36  [](Object& obj) -> std::string& {
37  return obj.name;
38  },
39  [](riku::variant obj, std::string& str) {
40  obj["name"].as<std::string>() = str;
41  }
42  )
43 )
44 ```

46 ### Sample usage
47 
48 ```cpp
49 Object obj;
50 
51 // Three ways to get type information, all return the same typeinfo.
52 riku::typeinfo obj_type = riku::inspect(obj),
53  by_type = riku::get<Object>(),
54  by_name = riku::get("Object");
55 
56 riku::variant allocated = riku::val(obj_type); // constructs a new default object if possible
57 
58 // variants can hold any documented type, either by copying the value or as a pointer
59 riku::variant as_val(obj); // copies obj
60 riku::variant as_ptr(&obj); // acts the same as as_val but will modify obj in-place.
61 riku::variant dbl_ptr(&&obj); // This allocates an Object* with the address of obj
62  // and will delete it when dbl_ptr leaves the stack
63 
64 // They can also be reassigned after the fact.
65 as_val = riku::ptr(obj); // Now it's the same as as_ptr and the memory it was holding is safely deleted.
66 as_val = riku::val(obj); // And it's a value again, with another independent copy.
67 
68 // variants are primarily used to access properties and functions in a generic fashion
69 as_ptr["children"] << new Object[5];
70 as_ptr["child_count"].as<unsigned>() = 5;
71 
72 as_ptr("GetChild", 3); // The same as obj.GetChild(3);
73 
74 // Custom properties look just like any other
75 as_val["ObjName"] << std::string("Test");
76 ```

82 Using RIKU in Your Project
83 ----------------------------------------
84 
85 ### Download
86 
87 ```bash
88 git clone /path/to/RIKU https://github.com/jwd-iv/RIKU
89 ```
90 
91 ### Build with CMake
92 
93 ```bash
94 cd /path/to/build/folder
95 cmake /path/to/RIKU
96 cmake /path/to/RIKU --build
97 ```
98 
99 ### Add to an existing CMake project
100 
101 ```cmake
102 add_subdirectory("/path/to/RIKU" "${CMAKE_CURRENT_BINARY_DIR}/RIKU_build")
103 
104 target_link_libraries(your_library RIKU)
105 ```
106 
107 ### Link statically
108 
109 Build and add RIKU.lib from the `build` folder to your project, and add `RIKU/include` to the list of include directories.
